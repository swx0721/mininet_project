# 校园网络仿真项目 — 技术名词解释

> 按功能模块分类，共 40+ 个技术名词

## 一、仿真平台 & 拓扑

### Mininet
轻量级网络仿真平台。在单台 Linux 主机上用虚拟化技术创建交换机、路由器、主机，每个虚拟节点运行独立的网络协议栈（真实 TCP/IP 内核实现），可以执行 ping、iperf、curl 等标准网络命令。比 GNS3/NS3 更轻量，适合教学和快速原型验证。

### LinuxRouter（Mininet 节点类型）
Mininet 内置的路由器节点类型，底层是一个启用了 IP 转发的 Linux 网络命名空间（namespace）。在我们的项目中，核心路由器 r1 就是一个 LinuxRouter，它负责六区域间的三层路由转发。

### 网络命名空间（Namespace）
Linux 内核提供的网络隔离机制。每个命名空间有独立的网卡、路由表、iptables 规则、ARP 表。Mininet 利用命名空间让每个虚拟主机拥有独立的网络栈。

### 接入交换机 / 汇聚交换机 / 核心路由器（三层架构）
- **接入交换机**：直接连接终端主机（如宿舍区的 PC），负责二层帧转发
- **汇聚交换机**：汇聚多台接入交换机的流量，向上连接到核心路由器
- **核心路由器**：跨子网的三层 IP 转发，所有安全策略部署于此

项目中的宿舍和教学楼采用了接入-汇聚二级交换 + 核心路由器的三层架构。

### Open vSwitch
Mininet 默认使用的虚拟交换机。支持 VLAN、流表、OpenFlow 协议。在我们的项目中用于所有二层交换节点。

### 子网（Subnet）
IP 地址的 CIDR 划分方式。例如 `10.0.0.0/20` 表示可用 IP 范围为 10.0.0.1 ~ 10.0.15.254，共 4094 个地址。子网越小（掩码越长），地址数越少，安全隔离性越强。

| 区域 | 子网 | 掩码含义 |
|------|------|---------|
| 宿舍区 | 10.0.0.0/20 | 大子网，容纳大量终端 |
| 财务处 | 10.0.35.0/26 | 小子网，仅 62 个可用地址 |
| 人事处 | 10.0.35.64/26 | 同上 |

## 二、路由 & 连通性

### IP 转发（IP Forwarding）
Linux 内核参数 `net.ipv4.ip_forward = 1`，允许路由器将收到的 IP 包从一块网卡转发到另一块网卡。不开启时路由器只处理发给自己 IP 的包。

### 路由表（Routing Table）
每台主机的内核维护的一张表，记录"目标网络 → 下一跳 IP + 出接口"的映射。Mininet 通过 `ip route add` 命令配置静态路由。

### PING（ICMP Echo/Reply）
网络可达性测试工具。发送 ICMP Echo Request 并等待 ICMP Echo Reply。如果收到 Reply 说明目标可达且路径通畅。在我们的项目中用于验证基础连通性。

### rp_filter（反向路径过滤）
Linux 内核安全参数。值为 1 时，内核会检查入站包的源 IP 是否可通过接收接口的反向路径到达，不匹配则丢弃。在某些 VPN/非对称路由场景下会导致合法包被误丢，需设为 0。

### sysctl
Linux 内核运行时参数配置工具。例如 `sysctl -w net.ipv4.ip_forward=1` 开启 IP 转发。修改立即生效，无需重启。

## 三、QoS 流量整形

### QoS（Quality of Service，服务质量）
在网络拥塞时有选择地保证关键业务的带宽和时延，而不是所有流量公平竞争。核心思想是"区分对待"——关键业务优先，普通业务让步。

### HTB（Hierarchy Token Bucket，分层令牌桶）
Linux 内核的一种排队规则（qdisc），支持多级带宽分配。工作方式是：给每个流量类别分配一个"令牌桶"，令牌以固定速率填充，发送数据需要消耗令牌。关键参数：
- **rate**（保障带宽）：无论多拥塞都能保证的最低速率
- **ceil**（上限带宽）：突发时可以借用的最大速率
- **prio**（优先级）：数值越小优先级越高，高优先级类先发送数据

在我们的项目中，财务处 rate=12Mbps、prio=0；其他区域 rate=2Mbps、prio=7。

### tc（Traffic Control）
Linux 流量控制框架。HTB、TBF 等排队规则都是通过 `tc qdisc` 和 `tc class` 命令配置的。

### 令牌桶（Token Bucket）
流量整形的数学模型。令牌以恒定速率产生，每个数据包需要消耗等量的令牌才能发送。如果令牌不足则排队等待。确保长期平均速率不超过设定值，同时允许短期突发。

### 吞吐量 / 时延 / 抖动 / 丢包率（四维网络指标）
- **吞吐量（Throughput）**：单位时间内实际成功传输的数据量，单位 Mbps
- **时延（Latency）**：数据包从源到目标所经历的时间，单位 ms
- **抖动（Jitter）**：时延的变化幅度（标准差），影响实时应用如 VoIP/视频
- **丢包率（Packet Loss）**：传输过程中丢失的包占比，拥塞时上升

## 四、负载均衡

### 负载均衡（Load Balancing）
将多台服务器的请求分散到不同的后端服务器，避免某一台过载而其他闲置。常用的调度算法有轮询、加权轮询、最少连接等。

### Round Robin（轮询算法）
最简单的负载均衡算法。维护一个计数器，每次请求到来时按顺序选择下一台服务器。例如 S1 → S2 → S1 → S2 ...。优点是实现简单、无状态，缺点是不同请求可能消耗不同资源，可能导致负载不均。

### Jain 公平指数
衡量资源分配公平性的指标，取值范围 [1/n, 1]（n 为参与者数量）。值越接近 1 越公平。公式：

$$\text{Jain}(x_1, ..., x_n) = \frac{(\sum x_i)^2}{n \cdot \sum x_i^2}$$

在我们的双服务器场景中，n=2。当两台服务器收到相同流量时 Jain=1.0，当一台满载另一台空闲时 Jain≈0.5。

### 负载方差
各服务器负载的统计方差。方差越大，负载越不均衡。与 Jain 指数互补——Jain 接近 1 时方差接近 0。

### iperf3
网络性能测量工具，可以生成 TCP/UDP 流并测量吞吐量、时延、抖动、丢包率。在我们的项目中用于所有消融实验的流量生成。支持客户端-服务器模式，一端监听（-s），一端发起（-c）。

## 五、安全策略

### ACL（Access Control List，访问控制列表）
基于 IP 地址/端口/协议的路由器包过滤规则。白名单明确允许，黑名单明确拒绝。在我们的项目中部署在 r1 的 FORWARD 链上。

### iptables / netfilter
Linux 内核防火墙框架。netfilter 是内核中的包过滤钩子，iptables 是用户态的规则管理工具。五个内置链：
- **INPUT**：处理进入本机的包
- **OUTPUT**：处理从本机发出的包
- **FORWARD**：处理经过本机转发的包（核心路由器场景最关键的链）
- **PREROUTING**：路由决策前（用于 DNAT）
- **POSTROUTING**：路由决策后（用于 SNAT/Masquerade）

### FORWARD 链
iptables 中处理**转发流量**（不发给本机、也不从本机发出）的规则链。在路由器场景中，所有跨区域流量都命中 FORWARD 链，因此所有 ACL 和 IDS 规则都部署在这里。

### 状态防火墙（Stateful Firewall）
基于连接状态的防火墙，不等同于简单的静态规则。通过 `-m state --state ESTABLISHED,RELATED` 匹配已建立的 TCP 连接和关联协议（FTP 数据通道等），允许回程包通过。优点是减少规则数量、提高效率。

### IDS（Intrusion Detection System，入侵检测系统）
实时监控网络流量，识别恶意行为（端口扫描、Flood 攻击等）并触发告警或自动响应。我们的项目实现了三类 IDS 检测：端口扫描、ICMP Flood、SYN Flood。

### ICMP Flood（Ping 洪泛攻击）
向目标发送大量 ICMP Echo Request（Ping）包，消耗目标的 CPU 和带宽资源。属于 DoS 攻击的一种。防护方式：iptyables limit 模块限速——1 个 ICMP 包/秒，超出部分 DROP。

### SYN Flood（TCP 半连接攻击）
发送大量 TCP SYN 包但不完成三次握手，耗尽目标的半连接队列（SYN backlog）。防护方式：iptables limit 模块限速——50 个 SYN 包/秒。

### iptables limit 模块
限速匹配模块，语法 `-m limit --limit 1/s --limit-burst 5`。limit 1/s 是长期平均速率，burst 5 是令牌桶初始容量（允许初始突发）。在我们的项目中用于 ICMP Flood（1/s）和 SYN Flood（50/s）防护。

### 滑动窗口（Sliding Window）
端口扫描检测的核心算法。维护一个 10 秒的时间窗口，统计窗口内某源 IP 访问的不同目标端口数。超过阈值 20 即判定为端口扫描行为，自动封禁 300 秒。

### 端口扫描（Port Scan）
攻击者向目标扫描大量端口以发现开放的服务。nmap 是经典工具。在我们的项目中用 Python socket 实现扫描模拟。

### 自动封禁（Auto-Ban）
检测到攻击行为后，自动通过 iptables 添加临时 DROP 规则，禁止攻击源 IP 的所有流量，持续 300 秒后自动解除。

## 六、审计 & 持久化

### SQLite
嵌入式关系型数据库，无需独立服务进程，数据存储为单个文件。在我们的项目中用于安全审计：记录每条 FLOOD、PORT_SCAN、BAN 事件的类型、时间戳、源 IP、详情。

### 审计（Audit）
记录安全事件的操作日志，支持事后溯源。谁、在什么时间、从哪个 IP、做了什么事、结果如何——五要素完整记录。

## 七、VPN 远程接入

### VPN（Virtual Private Network，虚拟专用网络）
在公共网络上建立加密隧道，让远程用户如同在本地一样访问内部网络资源。解决了远程接入的安全问题。

### WireGuard
新一代 VPN 隧道协议，2018 年并入 Linux 内核（5.6+）。特点：
- 代码量极小（约 4000 行），比 OpenVPN/IPsec 少一个数量级
- 使用 Curve25519 椭圆曲线密钥交换，安全性高
- 基于 UDP 传输，内置漫游支持
- 配置简单，没有复杂的证书体系

在我们的项目中，WireGuard 部署在 r1（服务端）和 home_pc（客户端）之间，隧道子网为 10.0.80.0/24。

### wg0（WireGuard 接口）
WireGuard 隧道在操作系统中表现为一个虚拟网络接口（如 `wg0`）。通过该接口发出的包会被加密并封装在 UDP 包中发送；收到的加密包解密后从该接口进入系统。在我们的 iptables 规则中通过 `-i wg0` 匹配 VPN 解密后的流量。

### Endpoint（WireGuard 端点）
WireGuard peer 配置中的远端地址和端口，形如 `192.168.100.1:51820`。客户端通过 endpoint 指向服务端的公网 IP 和监听端口。

### 公钥 / 私钥（Public Key / Private Key）
WireGuard 使用 Curve25519 非对称加密。每端生成一对密钥，公钥告诉对方，私钥保密。双方交换公钥后即可建立加密隧道。公钥格式为 base64 编码的 32 字节。

### UDP 51820
WireGuard 默认监听端口。服务端在 UDP 51820 等待客户端的加密包。协议无连接、无握手确认，只在有数据发送时才发包（静默模式下无流量）。

## 八、网络服务

### vsftpd
"Very Secure FTP Daemon"，Linux 系统上最常用的 FTP 服务器软件。在我们的项目中以 standalone 模式运行，匿名访问，托管在 server1 和 server2 上。

### FTP（File Transfer Protocol，文件传输协议）
标准的文件传输协议，使用 TCP 端口 21（控制连接）和数据端口（主动模式 20、被动模式随机）。在我们的项目中用于演示服务器文件的远程下载。

### HTTP（Hypertext Transfer Protocol）
网页传输协议，TCP 端口 80。在我们的项目中通过 Python SimpleHTTP 提供文件下载服务。

### curl
命令行 HTTP/FTP 客户端工具。支持 `curl http://...` 下载网页和 `curl ftp://...` 下载 FTP 文件。

### netcat（nc）
"TCP/IP 瑞士军刀"。可创建 TCP/UDP 连接、监听端口、传输数据。在我们的文件传输功能中，发送方用 `nc <ip> 9999 < file` 推送文件，接收方用 `nc -l 9999 > file` 接收。

## 九、实验方法论

### 消融实验（Ablation Study）
通过逐一移除（消融）系统的某个模块，对比消融前后的性能差异，量化该模块的独立贡献。类比医学中的对照实验。

我们的四个消融实验：
| 实验 | 对照组 | 实验组 | 目的 |
|------|--------|--------|------|
| QoS | Final - HTB | Final | 量化 HTB 优先级调度效果 |
| LB | Final - LB | Final | 量化 Round Robin 均衡效果 |
| Security | Final - Security | Final | 量化 ACL+IDS 安全效果 |
| Security Test | — | Final | 完整功能验证 |

### 对照组 / 实验组
- **对照组（Control Group）**：移除待测模块的基线版本（如 Final - HTB）
- **实验组（Experimental Group）**：包含完整模块的目标版本（如 Final）

两者的差异就是我们想要测量的模块效果。

### MD5 校验
文件完整性校验算法。发送方计算文件的 MD5 哈希值，接收方收到文件后重新计算——两者一致则文件传输无误；不一致说明数据损坏。

### TCP / UDP
- **TCP**（传输控制协议）：面向连接、可靠传输、有确认和重传机制，适合文件下载、网页浏览
- **UDP**（用户数据报协议）：无连接、不保证可靠、低开销，适合视频流、DNS 查询

在我们的实验中，财务处用 TCP（关键业务需要可靠传输），宿舍区用 UDP（模拟视频流场景）。

## 十、其他

### base64 编码
将二进制数据（如 WireGuard 密钥）转换为可打印 ASCII 字符串的编码方式。每 3 个字节编码为 4 个字符。

### 链路瓶颈（Bottleneck）
网络中带宽最小的那一段链路，决定了端到端的最大吞吐量。在我们的实验中，服务器上行链路的 20 Mbps 就是瓶颈。

### NAT（Network Address Translation，网络地址转换）
将私有 IP 地址转换为公网 IP 的技术。Mininet 默认对虚拟主机做 NAT 出公网，但在我们的 final 模型中被 `clear_all_rules` 清除，改为严格的 iptables 规则管理。
