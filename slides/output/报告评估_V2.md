#### 🔴 关键不一致 #1：QoS 实验数据不匹配

报告**表1**中的QoS消融实验数据与项目实际输出文件 `实验结果-新版统一各区域带宽上限.txt` 中的数据**完全不匹配**。

| 区域 | 报告表1 (Final-HTB消融) | 实测文件_场景A | 实测文件_场景B |
|------|------------------------|--------------|--------------|
| 财务处 TCP | 11.45 Mbps | 17.45 Mbps | 7.58 Mbps |
| 教学楼 TCP | 11.22 Mbps | 18.85 Mbps | 9.27 Mbps |
| 办公楼 TCP | 8.97 Mbps | 14.94 Mbps | 11.78 Mbps |
| 图书馆 TCP | 11.91 Mbps | 17.40 Mbps | 11.16 Mbps |
| 财务处 Final TCP | 15.87 Mbps | 18.52 Mbps | 17.01 Mbps |

**报告数据既不匹配场景A也不匹配场景B。** 这是一项严重的不一致。可能原因：
- 报告使用了第三次实验运行的数据（未保存到实验文件中）
- 报告数据是手动调整或估算的
- 实验参数不同导致结果差异

**建议**：必须确认报告 Table 1 的数据来源，用实际运行结果替换，或注明为哪次实验的数据。

#### 🟡 轻微不一致 #2：HTB作用位置描述

报告说："QoS策略通过分层令牌桶机制在核心路由器的服务器出口链路上实施带宽优先级管理。"——这是正确的。

但报告又说："HTB（Hierarchy Token Bucket）在区域交换机至核心路由器的上行链路部署 HTB 两级优先级策略"。这里的"区域交换机至核心路由器的上行链路"乍一看容易以为是在各区域上行链路，实际上指的是服务器出口链路。代码注释 `qos.py:5` 写了"实验瓶颈在服务器出口（默认 20Mbps），QoS 配在此处控制竞争"——这点一致，但报告措辞可能引起误解。

#### 🟡 轻微不一致 #3：home_pc 链路带宽

报告说"所有链路带宽统一设置为100Mbps"，但 `topology.py:132` 中 home_pc 的连接链路为 `bw=50, delay=20ms`（模拟互联网）。`extended_topology.py:126-136` 也有不同带宽的链路。建议改为"校园网内部链路统一为100Mbps"。

## 三、AI 味分析

### 🔴 AI味偏重的部分

**1. 文献综述段落（实现原理及方案 > 第2-4段）**

```
"在QoS机制方面，现有研究主要围绕链路调度与带宽保障展开，常见方法包括基于队列调度的分层令牌桶..."
"在负载均衡方面，常见调度策略包括轮询、最少连接数以及基于响应时间的动态加权调度..."
```

这几段是典型的AI生成式文献综述风格：
- 格式高度模板化："在X方面，常见Y方法包括A、B、C。其中A有M优点但N缺点，B在P场景下更优但Q不足，C则"
- 参考文献[1]-[14]的引用密度异常高（14篇参考文献，课程项目报告不需要这么密集）
- 每个技术点都有"现有研究主要围绕..."这种总括句

**判断**：这几段大概率是AI辅助生成后手动修改的。核心问题是**罗列知识多，结合项目少**——比如引用了WFQ、PQ、LC、WRT等并未在项目中实际使用的算法，占用了篇幅但没有增加信息量。

**建议**：大幅精简文献综述部分（从约800字缩减到约200字），只保留与项目选择直接相关的HTB和RR的简要背景，删除PQ/WFQ/LC/WRT等未实现算法的铺陈。

**2. 项目要求段中的长难句**

```
"网络安全方面，针对校园网中敏感区域保护及外部威胁防御需求，构建基于WireGuard协议的虚拟专用网络..."
```

这类句子是AI生成中文的典型特征——一句话塞入5个以上的技术点，读起来累。

### 🟢 AI味较轻的部分

**实现过程 > 安全策略 > VPN接入**: VPN的阶段描述（服务端初始化→隧道建立→访问控制）结合了项目的具体细节（r1-eth8、wg0接口、10.0.80.10等），读起来不像纯AI生成。

**结果分析 > QoS/负载均衡消融实验**: 数据分析和解释自然，有"礼堂"、"增福"等分析视角，这些需要人工判断。

**项目总结 > 主要困难与解决方案**: 提到"将HTB迁移至服务器出口链路"、"将拓扑修改为双独立上行链路架构"、"两套规则之间发生了干扰"——这些都是真实的开发经历，AI无法凭空编造。

### AI味总体判断

报告的AI味呈现**两端重、中间轻**的分布：
- 前端（实现原理、文献综述）：AI味重，占约15%
- 中端（实现过程、结果分析）：AI味轻，有项目特色
- 后端（困难/总结/参考文献）：适中，参考文献列表AI味明显
