## 1. 基础连通性测试

### 1.1 二层通信（区域内）

```bash
# 启动 Final 模型并进入 CLI
sudo python3 main.py --model final

# Mininet CLI 内执行：
dorm1 ping -c 3 dorm2          # 宿舍区两台主机互通（二层）
teach1 ping -c 3 teach2        # 教学楼两台主机互通（二层）
finance1 ping -c 3 finance2    # 财务处两台主机互通（二层）
hr1 ping -c 3 hr2              # 人事处两台主机互通（二层）
office1 ping -c 3 office2      # 办公楼两台主机互通（二层）
lib1 ping -c 3 lib2            # 图书馆两台主机互通（二层）
```

**预期结果：** 所有 ping 通，丢包率 0%。

### 1.2 三层通信（区域间）

```bash
# Mininet CLI 内执行：
dorm1 ping -c 3 teach1         # 宿舍区 → 教学楼（三层，经 r1 转发）
teach1 ping -c 3 office1       # 教学楼 → 办公楼
office1 ping -c 3 finance1     # 办公楼 → 财务处（白名单放行，应通）
dorm1 ping -c 3 finance1       # 宿舍区 → 财务处（黑名单拦截，不通）
```

**预期结果：**
- dorm1 → teach1: 通（三层路由可达）
- office1 → finance1: 通（ACL 白名单放行）
- dorm1 → finance1: **不通**（ACL 黑名单 DROP）

### 1.3 HTTP 访问测试

```bash
# Mininet CLI 内执行：
dorm1 curl http://10.0.60.2/           # 访问 Server1 主页
teach1 curl http://10.0.60.2/          # 教学楼访问 Server1
office1 curl http://10.0.60.18/        # 办公楼访问 Server2
```

**预期结果：** 返回校园网 HTML 页面，HTTP 200。

### 1.4 FTP 访问测试

```bash
# Mininet CLI 内执行：
dorm1 curl ftp://10.0.60.2/            # FTP 列出文件
```

**预期结果：** 返回 FTP 服务器文件列表（README.txt, share_doc.txt, notices/）。

### 1.5 iperf3 带宽测试

```bash
# Mininet CLI 内执行：
# 先在 Server1 上手动启动 iperf3 服务端：
server1 iperf3 -s -p 5201 -D

# 客户端发起流量测试：
dorm1 iperf3 -c 10.0.60.2 -p 5201 -t 10
```

**预期结果：** 显示带宽测试结果（受 20 Mbps 瓶颈限制）。

### 1.6 Interactive Network Communication Framework（交互式网络通信）

CLI 扩展命令，在 Mininet> 环境中动态发起任意主机之间的文件传输、消息通信和路径追踪。

**加载方式（单条 py 命令，用 tuple 确保同作用域）：**

```bash
# Mininet CLI 内执行：
py (exec(open('network_cli.py').read(), globals()), register(net))[1]
```

**命令清单：**

```bash
# 文件传输（经过真实路由+ACL检查+QoS模拟延迟）
send <src_host> <dst_host> <filename>

# 文本消息通信（TCP/Netcat 模拟）
msg <src_host> <dst_host> <message>

# 路径追踪（显示完整路由路径+ACL判定）
trace <src_host> <dst_host>

# 查看主机 inbox/outbox
ls <host_name>

# 列出可用资源文件
resources
```

**示例：**

```bash
# 宿舍区内部文件传输（二层，经 s_dorm1 转发）
send dorm1 dorm2 campus.jpg
# 预期: [ROUTE] dorm1 → s_dorm1 → dorm2 | [OK] 传输完成

# 跨区域文件传输（三层，经 r1 转发 + ACL 检查）
send office1 dorm1 notice.docx
# 预期: [ROUTE] office1 → s_office → r1 → sd_agg → s_dorm1 → dorm1

# 敏感区域访问拦截
send dorm1 finance1 campus.jpg
# 预期: [ACL] ACL_DENY (非授权区域禁止访问敏感区域) — 传输被拦截

# VPN 用户访问敏感区域拦截
send home_pc finance1 campus.jpg
# 预期: [ACL] ACL_VPN_DENY (VPN用户禁止访问敏感区域) — 传输被拦截

# 文本消息
msg office1 dorm1 "Meeting at 3pm"
# 预期: [MSG] office1 → dorm1: "Meeting at 3pm" | 已送达

# 路径追踪（含双校区）
trace dorm1 campusb_h1
# 预期: dorm1 → s_dorm1 → sd_agg → r1 → wan_rt → campusb_rt → s_campusb → campusb_h1

# 查看收件箱
ls dorm1
# 预期: [dorm1] inbox/ campus.jpg (1024 bytes) | outbox/ (empty)
```

**文件系统结构：**

启动时自动生成：
```
results/fs_topology/nodes/
├── dorm1/inbox/ outbox/
├── office1/inbox/ outbox/
├── finance1/inbox/ outbox/
├── ...
```

**传输日志**：每笔传输记录到 `results/runtime_transfer.json`，包含 src/dst/file/size/path/acl_result/qos_delay_ms/transfer_time/md5_match 字段。

## 6. ACL 防 VPN 绕过实验

### 6.1 运行实验

```bash
sudo python3 main.py --experiment acl_vpn_test
```

### 6.2 实验设计

| 阶段 | ACL 类型 | VPN→Finance | VPN→HTTP |
|------|---------|-------------|----------|
| 对照组 | 原始 ACL (仅基于源 IP) | 可能绕过 | 正常 |
| 实验组 | VPN 增强 ACL (身份感知) | 已拦截 | 正常 |

### 6.3 预期结果

- 对照组 VPN→Finance：**ALLOWED**（漏洞验证）
- 实验组 VPN→Finance：**BLOCKED**（修复验证）
- 实验组 VPN→HTTP：**ALLOWED**（正常功能不受影响）

## 附录：手动交互测试指令

以下指令在 `sudo python3 main.py --model final` 进入 Mininet CLI 后使用。

### A1. 区域间连通性

```bash
# 二层通信（同区域）
dorm1 ping -c 3 dorm2
teach1 ping -c 3 teach2
finance1 ping -c 3 finance2

# 三层通信（跨区域）
dorm1 ping -c 3 teach1
office1 ping -c 3 finance1
dorm1 ping -c 3 office1
```

### A2. Web/FTP 访问

```bash
# HTTP
dorm1 curl -s http://10.0.60.2/
teach1 curl -s http://10.0.60.2/

# FTP
dorm1 curl -s ftp://10.0.60.2/
office1 curl -s ftp://10.0.60.2/

# 大文件下载测试
dorm1 curl -o /dev/null -w "Speed: %{speed_download}\n" http://10.0.60.2/bigfile.bin
```

### A3. iperf3 手动测试

```bash
# 服务端（Server1）
server1 iperf3 -s -p 5201 -D

# TCP 测试
dorm1 iperf3 -c 10.0.60.2 -p 5201 -t 10
teach1 iperf3 -c 10.0.60.2 -p 5202 -t 10

# UDP 测试
dorm1 iperf3 -c 10.0.60.2 -p 5204 -u -b 12M -t 10
```

### A4. ACL 手动验证

```bash
# 白名单验证（应通过）
office1 ping -c 3 10.0.35.2      # office → finance
office1 ping -c 3 10.0.35.66     # office → hr

# 黑名单验证（应被拦截）
dorm1 ping -c 3 10.0.35.2        # dorm → finance
teach1 ping -c 3 10.0.35.2       # teach → finance
lib1 ping -c 3 10.0.35.2         # lib → finance
dorm1 ping -c 3 10.0.35.66       # dorm → hr
```

### A5. VPN 手动测试（启动扩展拓扑后）

```bash
# 校外主机访问校园网（VPN 隧道激活）
home_pc ping -c 3 10.0.60.2      # VPN → Server1 (应通)
home_pc curl -s http://10.0.60.2/ # VPN → HTTP (应返回页面)
home_pc ping -c 3 10.0.35.2      # VPN → 财务处 (应被拒绝)
home_pc ping -c 3 10.0.35.66     # VPN → 人事处 (应被拒绝)
```

### A6. NAT 手动测试

```bash
# NAT OFF 状态
inet_rt iptables -t nat -F        # 清除 NAT 规则
dorm1 ping -c 3 203.0.113.10      # 应 100% loss

# NAT ON 状态
inet_rt iptables -t nat -A POSTROUTING -s 10.0.0.0/16 -o inet_rt-eth2 -j MASQUERADE
dorm1 ping -c 3 203.0.113.10      # 应成功
```

### A7. 跨校区手动测试

