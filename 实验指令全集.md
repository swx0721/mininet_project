## 1. 基础连通性测试

### 1.1 二层通信（区域内）

```bash
# 启动 Final 模型并进入 CLI
sudo python3 main.py --model final

# Mininet CLI 内执行：
dorm1 ping -c 3 dorm2          # 宿舍区两台主机互通（二层）
teach1 ping -c 3 teach2        # 教学楼两台主机互通（二层）
finance1 ping -c 3 finance2    # 财务处两台主机互通（二层）
hr1 ping -c 3 hr2              # 人事处两台主机互通（二层）
office1 ping -c 3 office2      # 办公楼两台主机互通（二层）
lib1 ping -c 3 lib2            # 图书馆两台主机互通（二层）
```

**预期结果：** 所有 ping 通，丢包率 0%。

### 1.2 三层通信（区域间）

```bash
# Mininet CLI 内执行：
dorm1 ping -c 3 teach1         # 宿舍区 → 教学楼（三层，经 r1 转发）
teach1 ping -c 3 office1       # 教学楼 → 办公楼
office1 ping -c 3 finance1     # 办公楼 → 财务处（白名单放行，应通）
dorm1 ping -c 3 finance1       # 宿舍区 → 财务处（黑名单拦截，不通）
```

**预期结果：**
- dorm1 → teach1: 通（三层路由可达）
- office1 → finance1: 通（ACL 白名单放行）
- dorm1 → finance1: **不通**（ACL 黑名单 DROP）

### 1.3 HTTP 访问测试

**预期结果：** 返回校园网 HTML 页面，HTTP 200。

### 1.4 FTP 访问测试

### 1.6 Interactive Network Communication Framework（交互式网络通信）

# 启动 Final 模型并进入 CLI
sudo python3 main.py --model final

CLI 扩展命令，在 Mininet> 环境中动态发起任意主机之间的文件传输、消息通信和路径追踪。

**加载方式（单条 py 命令，用 tuple 确保同作用域）：**

```bash
# Mininet CLI 内执行：
py (exec(open('network_cli.py').read(), globals()), register(net))[1]
```

**命令清单：**

```bash
# 文件传输（经过真实路由+ACL检查+QoS模拟延迟）
send <src_host> <dst_host> <filename>

# 文本消息通信（TCP/Netcat 模拟）
msg <src_host> <dst_host> <message>

# 路径追踪（显示完整路由路径+ACL判定）
trace <src_host> <dst_host>

# 查看主机 files/
ls <host_name>

# 列出可用资源文件
resources
```

**示例：**

```bash
# 宿舍区内部文件传输（二层，经 s_dorm1 转发）
send dorm1 dorm2 campus.jpg
# 预期: [ROUTE] dorm1 → s_dorm1 → dorm2 | [OK] 传输完成

# 跨区域文件传输（三层，经 r1 转发 + ACL 检查）
send office1 dorm1 通知_2026.docx
# 预期: [ROUTE] office1 → s_office → r1 → sd_agg → s_dorm1 → dorm1

#白名单测试：
send office1 finance1 通知_2026.docx

# 从服务器下载文件（dorm1 从 server1 拉取 zip 压缩包）
send server1 dorm1 DMS_essay.zip
# 预期: [ROUTE] server1 → s_server1 → r1 → sd_agg → s_dorm1 → dorm1
# [OK] 传输完成! MD5: ✓ 一致

# 敏感区域访问拦截
send dorm1 finance1 campus.jpg
# 预期: [ACL] ACL_DENY (非授权区域禁止访问敏感区域) — 传输被拦截

# ============ VPN 两层 ACL 演示 ============

# VPN 关闭时：校外 home_pc 完全隔离，无法访问任何校内主机
send home_pc dorm1 campus.jpg
send dorm1 home_pc campus.jpg
# 预期: [ERROR] ping 不通 / 传输超时（iptables 全局 DROP）

# VPN 开启后：home_pc 获得校内普通用户权限，可访问普通区域
py enable_vpn("home_pc")
send home_pc dorm1 campus.jpg
send home_pc dorm1 VPN测试文件.pdf
# 预期: [ROUTE] home_pc → s_home → r1 → sd_agg → s_dorm1 → dorm1 | [OK] 传输完成

# 但 VPN 用户仍禁止访问敏感区域
send home_pc finance1 campus.jpg
# 预期: [ACL] ACL_VPN_DENY (VPN用户禁止访问敏感区域) — 传输被拦截

# 断开 VPN 恢复隔离
py disable_vpn("home_pc")
send home_pc dorm1 campus.jpg
# 预期: 再次超时（恢复全局 DROP）

# 文本消息
msg office1 dorm1 "Meeting at 3pm"
msg office1 dorm2 "Meeting at 3pm"
# 预期: [MSG] office1 → dorm1: "Meeting at 3pm" | 已送达

# 路径追踪（含双校区）
trace dorm1 campusb_h1
# 预期: dorm1 → s_dorm1 → sd_agg → r1 → wan_rt → campusb_rt → s_campusb → campusb_h1

# 追踪 VPN 路径
trace home_pc dorm1
# 预期: home_pc → s_home → r1 → sd_agg → s_dorm1 → dorm1

# 查看文件目录
ls dorm1
# 预期: [dorm1] files/ campus.jpg (xxx bytes) | poster.jpg (xxx bytes)
```

**文件系统结构：**

启动时自动生成：
```
fs_topology/nodes/
├── dorm1/files/
├── office1/files/
├── finance1/files/
├── ...
```

**传输日志**：每笔传输记录到 `fs_topology/runtime_transfer.json`，包含 src/dst/file/size/path/acl_result/qos_delay_ms/transfer_time/md5_match 字段。

### 1.7 Interactive 交互子系统（统一 py 调用）

除 `send`/`msg`/`trace`/`ls` 等 CLI 命令外，`interactive/` 提供了更丰富的 `py` 函数调用接口，适合答辩现场演示。

**加载方式：**

```bash
# Mininet 不支持 from X import * 语法，用 setup() 一键加载：
mininet> py import interactive; interactive.setup()
# 输出: [INTERACTIVE] 14 个交互函数已就绪
```

> **注：两套交互系统说明**
>
> | 特性 | 1.6 `network_cli.py`（CLI 命令） | 1.7 `interactive/`（py 函数） |
> |------|----------------------------------|-------------------------------|
> | 入口 | `py (exec(...), register(net))[1]` | `py import interactive; interactive.setup()` |
> | 文件传输 | `send dorm1 dorm2 campus.jpg` | `transfer_file("dorm1","dorm2","campus.jpg")` |
> | 消息 | `msg dorm1 dorm2 "hello"` | 无独立命令（用 1.6） |
> | VPN 开关 | 无（需配合 1.7 的 enable_vpn） | `enable_vpn("home_pc")` / `disable_vpn("home_pc")` |
> | 端口扫描 | 不支持 | `scan("dorm1","server1")` |
> | Flood 攻击 | 不支持 | `flood("dorm1","server1",40)` |
> | 身份查看 | 不支持 | `show_identity("home_pc")` |
> | HTTP 下载 | 不支持（用 send 代替） | `download_http("dorm1","campus.jpg")` |
>
> **推荐用法**：答辩演示时，先加载 1.6（CLI 快速操作），再加载 1.7（VPN/安全演示）。两者可同时使用。

**命令清单：**

| 函数 | 用法 | 说明 |
|------|------|------|
| `transfer_file` | `transfer_file("dorm1","dorm2","campus.jpg")` | 文件传输 + MD5 + 吞吐统计 |
| `show_identity` | `show_identity("home_pc")` | 查看主机身份 (STUDENT/STAFF/VPN_USER...) |
| `switch_identity` | `switch_identity("home_pc","STAFF")` | 切换身份（模拟权限变更） |
| `enable_vpn` | `enable_vpn("home_pc")` | 启用 VPN（修改 iptables：全局隔离→VPN ACL） |
| `disable_vpn` | `disable_vpn("home_pc")` | 断开 VPN（恢复 iptables 全局隔离） |
| `download_http` | `download_http("dorm1","课件下载.zip")` | HTTP 下载（自动保存 files + ACL 检查） |
| `scan` | `scan("dorm1","server1")` | 端口扫描攻击 → 触发 IDS 封禁 |
| `show_blacklist` | `show_blacklist()` | 查看封禁 IP 列表 |
| `show_conntrack` | `show_conntrack()` | 查看状态防火墙连接追踪 |
| `flood` | `flood("dorm1","server1",40)` | ICMP Flood 攻击 → iptables limit 防护 |
| `access_resource` | `r=access_resource("home_pc","课件.zip"); print_access_result(r)` | 统一双层 ACL 检查入口 |

### 1.8 双层 ACL 访问控制演示 (Perimeter + Internal)

**加载方式：**

```bash
mininet> py from security.perimeter_acl import *
```

**三层信任网络：** `10.0.0.0/16` (Campus-A) + `10.1.0.0/16` (Campus-B) + `10.0.80.0/24` (VPN)

**演示流程：**

```bash
mininet> py show_identity("home_pc")
# IP: 192.168.100.10 / Identity: HOME_USER

# 校外主机 → 边界拒绝
mininet> py download_http("home_pc","课件下载.zip")
# ACCESS DENIED — Perimeter ACL (源地址不在校园网络边界内)

# 启用 VPN → 获得信任 IP
mininet> py enable_vpn("home_pc")
# IP: 10.0.80.10 / Identity: VPN_USER

# VPN用户访问普通课件 → 成功
mininet> py download_http("home_pc","课件下载.zip")
# ACCESS GRANTED

# VPN用户访问财务数据 → 内部ACL拒绝
mininet> py download_http("home_pc","财务数据报表..xlsx")
# ACCESS DENIED — Campus Internal ACL (VPN_USER 无权访问敏感资源)

# 只有 FINANCE_ADMIN 才能访问
mininet> py download_http("finance1","财务数据报表..xlsx")
# ACCESS GRANTED
```

### 1.9 安全防御实时演示

**端口扫描 → IDS 自动封禁：**

```bash
mininet> py scan("dorm1","server1")

# Port 20: PORT SCAN DETECTED — Blocked for 300 seconds

mininet> py show_blacklist()
# 10.0.0.2  剩余 298s
```

**ICMP Flood → iptables limit 防护：**

```bash
mininet> py flood("dorm1","server1",40)

```
