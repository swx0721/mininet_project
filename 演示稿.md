### 一、启动网络环境

首先启动校园网络仿真系统。

执行命令：

```bash
sudo python3 main.py --model final
```

该命令会自动构建整个校园网络拓扑，包括宿舍区、办公区、图书馆、财务处、服务器区以及校外网络等节点，同时加载ACL访问控制、QoS流量管理、负载均衡、入侵检测和安全审计等功能模块。

当界面出现 `mininet>` 提示符时，说明网络环境已经成功启动。

---

### 二、加载交互式通信模块

接下来加载交互式通信功能。

执行命令：

```python
py (exec(open('network_cli.py').read(), globals()), register(net))[1]
```

该命令用于扩展Mininet命令行功能，为系统增加文件传输、消息通信以及文件查看等自定义指令。

加载成功后，我们就可以使用：

* send：文件传输
* msg：消息发送
* ls：查看节点文件

等功能进行后续演示。

---

### 三、基础网络连通性测试

首先测试宿舍区内部通信。

执行：

```bash
dorm1 ping -c 3 dorm2
```

这里模拟宿舍主机 dorm1 与 dorm2 之间的通信。

由于二者位于同一区域，因此数据包可以正常到达目标主机，通信成功。

---

接下来测试跨区域访问。

执行：

```bash
dorm1 ping -c 3 finance2
```

这里模拟宿舍区访问财务处。

根据系统预设的访问控制策略，宿舍区不允许直接访问财务处等敏感区域，因此该请求会被ACL规则拦截。

通过这个实验可以验证访问控制策略已经生效。

---

### 四、应用层服务访问测试

首先访问Web服务器。

执行：

```bash
dorm1 curl http://10.0.60.2/
```

该命令模拟宿舍区用户访问校园服务器提供的网站服务。

如果返回网页内容，说明：

* 网络路由正常；
* ACL策略允许访问；
* Web服务运行正常。

---

然后测试FTP服务。

执行：

```bash
lib1 curl ftp://10.0.60.2/
```

这里模拟图书馆用户访问服务器上的FTP服务。

如果能够成功返回FTP欢迎信息，则说明FTP服务已经正常启动并能够被客户端访问。

---

### 五、文件传输与消息通信演示

首先演示文件传输功能。

执行：

```bash
send dorm1 dorm2 campus.jpg
```

该命令表示将图片文件 campus.jpg 从 dorm1 发送到 dorm2。

系统会自动完成：

* 访问权限检查；
* 路径选择；
* 文件传输；
* 完整性校验；
* 日志记录。

当显示传输完成时，说明文件已经成功到达目标主机。

---

接下来演示文本消息发送。

执行：

```bash
msg office1 dorm2 "Meeting at 3pm."
```

该命令表示办公区主机 office1 向 dorm2 发送一条通知消息。

系统会自动生成消息文件并保存到目标主机目录中，实现类似校园内部通知系统的功能。

---

随后查看节点文件。

执行：

```bash
ls dorm1
```

以及：

```bash
ls dorm2
```

通过查看文件列表，可以验证文件和消息是否已经成功送达目标节点。

### 六、VPN功能演示

下面演示校外用户通过VPN访问校园网络的过程。

这里的 home_pc 代表校外主机。

---

#### （1）VPN关闭状态

首先保持VPN关闭。

执行：

```bash
send home_pc dorm1 VPN测试文件.pdf
```

以及：

```bash
send home_pc finance1 VPN测试文件.pdf
```

此时校外主机尚未接入VPN，因此无法访问校园网络。

两次传输都会被系统直接阻止。

这体现了校园网络对外部访问的隔离机制。

---

#### （2）开启VPN

执行：

```python
py enable_vpn("home_pc")
```

该命令为校外主机建立VPN连接。

VPN建立后，校外用户获得访问校园网络的权限。

随后再次执行：

```bash
send home_pc dorm1 VPN测试文件.pdf
```

此时文件能够成功传输到宿舍区主机。

说明VPN连接已经生效。

---

然后继续执行：

```bash
send home_pc finance1 VPN测试文件.pdf
```

虽然VPN已经开启，但财务处属于敏感区域。

系统仍然会根据访问控制策略拒绝该请求。

这说明VPN并不是完全开放权限，而是在认证成功后仍然受到校园安全策略的约束。

---

#### （3）关闭VPN

执行：

```python
py disable_vpn("home_pc")
```

关闭VPN连接。

随后再次执行：

```bash
send home_pc dorm1 VPN测试文件.pdf
```

以及：

```bash
send home_pc finance1 VPN测试文件.pdf
```

两次请求都会再次被阻止。

说明VPN关闭后，校外主机重新回到隔离状态。

---

通过以上演示，我们验证了校园网络系统的多个核心功能：
